# IEEE Standards Implementation Guide - Technical Reference\n\n## Overview\nThis guide documents how to implement IEEE academic standards compliance in research websites.\n\n---\n\n## 1. Metadata Implementation\n\n### 1.1 Essential Meta Tags (Minimum)\n\n`html\n<!-- Basic Research Identification -->\n<meta name=\"description\" content=\"Your research project description\"/>\n<meta name=\"keywords\" content=\"keyword1, keyword2, keyword3\"/>\n<meta name=\"author\" content=\"Author Name\"/>\n<meta name=\"institution\" content=\"Institution Name\"/>\n<meta name=\"project-id\" content=\"PROJECT_CODE\"/>\n<meta name=\"version\" content=\"1.0.0\"/>\n<meta name=\"published-date\" content=\"YYYY-MM-DD\"/>\n<meta name=\"last-modified\" content=\"YYYY-MM-DD\"/>\n`\n\n### 1.2 Dublin Core Metadata (Recommended)\n\n`html\n<!-- Required Dublin Core Elements -->\n<meta name=\"DC.title\" content=\"Full project title\"/>\n<meta name=\"DC.creator\" content=\"Author1, Author2\"/>\n<meta name=\"DC.date\" content=\"2025-09-08\"/>\n<meta name=\"DC.issued\" content=\"2025-09-08\"/>\n<meta name=\"DC.modified\" content=\"2026-04-27\"/>\n<meta name=\"DC.identifier\" content=\"urn:uuid:unique-identifier\"/>\n<meta name=\"DC.type\" content=\"Research Project\"/>\n<meta name=\"DC.subject\" content=\"Keyword1; Keyword2; Keyword3\"/>\n<meta name=\"DC.coverage\" content=\"Geographic scope\"/>\n<meta name=\"DC.rights\" content=\"License type and URL\"/>\n`\n\n### 1.3 Citation Metadata\n\n`html\n<!-- Google Scholar & Citation Tools -->\n<meta name=\"citation_title\" content=\"Full title\"/>\n<meta name=\"citation_authors\" content=\"Author1, Author2\"/>\n<meta name=\"citation_publication_date\" content=\"YYYY/MM/DD\"/>\n<meta name=\"citation_institution\" content=\"Institution\"/>\n<meta name=\"citation_keywords\" content=\"keyword1, keyword2\"/>\n<meta name=\"citation_abstract\" content=\"Project abstract\"/>\n<meta name=\"citation_doi\" content=\"10.xxxx/xxxxx\"/>\n<meta name=\"citation_url\" content=\"https://example.com\"/>\n<meta name=\"citation_project_code\" content=\"CODE_123\"/>\n`\n\n### 1.4 IEEE-Specific Format\n\n`html\n<!-- Pre-formatted IEEE Citation -->\n<meta name=\"citation_ieee_style\" content=\"Author, A., Author, B. (Year). Project title. Institution, Project Code. DOI: xxxxx\"/>\n`\n\n---\n\n## 2. JSON-LD Structured Data\n\n### 2.1 Basic JSON-LD Template\n\n`html\n<script type=\"application/ld+json\">\n{\n  \"@context\": \"https://schema.org\",\n  \"@type\": \"ResearchProject\",\n  \"name\": \"Project title\",\n  \"identifier\": \"PROJECT_CODE\",\n  \"description\": \"Detailed description\",\n  \"url\": \"https://example.com\",\n  \"creator\": [\n    {\n      \"@type\": \"Person\",\n      \"name\": \"Author Name\",\n      \"identifier\": \"Student_ID\",\n      \"email\": \"email@example.com\"\n    }\n  ],\n  \"datePublished\": \"2025-09-08\",\n  \"dateModified\": \"2026-04-27\",\n  \"inLanguage\": \"en\",\n  \"isAccessibleForFree\": true,\n  \"keywords\": \"keyword1, keyword2, keyword3\",\n  \"license\": \"https://creativecommons.org/licenses/by/4.0/\",\n  \"sponsor\": {\n    \"@type\": \"Organization\",\n    \"name\": \"Organization Name\",\n    \"url\": \"https://example.org\"\n  },\n  \"mentor\": [\n    {\n      \"@type\": \"Person\",\n      \"name\": \"Supervisor Name\"\n    }\n  ]\n}\n</script>\n`\n\n### 2.2 Validation\n\n- **Validator:** https://validator.schema.org/\n- **Steps:**\n 1. Copy JSON-LD block\n 2. Paste into validator\n 3. Check for errors\n 4. Fix any validation issues\n\n---\n\n## 3. DOI Implementation\n\n### 3.1 Obtaining a DOI\n\n**Option 1: OSF (Recommended for Student Projects)**\n`\n1. Go to: https://osf.io/\n2. Create account\n3. Create project\n4. Upload files\n5. System auto-generates DOI\n6. Format: 10.31219/osf.io/XXXXX\n`\n\n**Option 2: Zenodo**\n`\n1. Go to: https://zenodo.org/\n2. Create account\n3. Upload research\n4. System auto-generates DOI\n5. Format: 10.5281/zenodo.XXXXX\n`\n\n**Option 3: CrossRef (Professional)**\n`\n- More complex\n- Requires institutional membership\n- Better for published papers\n`\n\n### 3.2 Using DOI in Metadata\n\n`html\n<!-- Meta tag -->\n<meta name=\"citation_doi\" content=\"10.31219/osf.io/123456\"/>\n\n<!-- Resolvable link -->\n<a href=\"https://doi.org/10.31219/osf.io/123456\">\n  https://doi.org/10.31219/osf.io/123456\n</a>\n\n<!-- In citation -->\nDOI: 10.31219/osf.io/123456\n`\n\n---\n\n## 4. WCAG 2.1 Level AA Implementation\n\n### 4.1 Skip Navigation Links\n\n`html\n<!-- Add at start of body -->\n<a href=\"#main-content\" class=\"skip-link\">\n  Skip to main content\n</a>\n\n<style>\n.skip-link {\n  position: absolute;\n  top: -40px;\n  left: 0;\n  background: #000;\n  color: #fff;\n  padding: 0.5rem;\n  text-decoration: none;\n  z-index: 100;\n}\n\n.skip-link:focus {\n  top: 0; /* Becomes visible on focus */\n}\n</style>\n`\n\n### 4.2 ARIA Labels\n\n`html\n<!-- Navigation -->\n<nav role=\"navigation\" aria-label=\"Main navigation\">\n  <ul role=\"menubar\">\n    <li><a href=\"#\" role=\"menuitem\">Link</a></li>\n  </ul>\n</nav>\n\n<!-- Sections -->\n<section role=\"region\" aria-label=\"Research overview\">\n  <h2>Content</h2>\n</section>\n\n<!-- Buttons -->\n<button aria-label=\"Toggle menu\" aria-expanded=\"false\">\n  Menu\n</button>\n\n<!-- Images -->\n<img src=\"image.jpg\" alt=\"Descriptive alt text\" title=\"Tooltip text\"/>\n\n<!-- Form -->\n<form>\n  <label for=\"name\">Name:</label>\n  <input id=\"name\" type=\"text\" aria-required=\"true\"/>\n</form>\n`\n\n### 4.3 Semantic HTML\n\n`html\n<!-- Correct semantic structure -->\n<body role=\"document\">\n  <nav role=\"navigation\">...</nav>\n  \n  <main>\n    <article>\n      <h1>Main heading</h1>\n      <h2>Section heading</h2>\n      <p>Content</p>\n    </article>\n  </main>\n  \n  <footer role=\"contentinfo\">...</footer>\n</body>\n`\n\n### 4.4 Color Contrast (WCAG AA)\n\n`\nMinimum ratios:\n- Regular text: 4.5:1 contrast ratio\n- Large text (18pt+): 3:1 contrast ratio\n\nCheck with: https://contrast-ratio.com/\n\nExample:\n- Text: #111827 (dark)\n- Background: #ffffff (white)\n- Ratio: 12.63:1 ✓ Passes\n`\n\n### 4.5 Keyboard Navigation\n\n`html\n<!-- Ensure all interactive elements are keyboard accessible -->\n<a href=\"#\">Link</a> <!-- Naturally keyboard accessible -->\n<button>Button</button> <!-- Naturally keyboard accessible -->\n<input type=\"text\"/> <!-- Naturally keyboard accessible -->\n\n<!-- Make custom elements keyboard accessible -->\n<div role=\"button\" \n     tabindex=\"0\" \n     onkeypress=\"handleKeyPress(event)\"\n     onclick=\"handleClick()\">\n  Custom button\n</div>\n`\n\n---\n\n## 5. Accessibility Statement\n\n### 5.1 HTML Structure\n\n`html\n<section id=\"accessibility-statement\">\n  <h2>Accessibility Statement</h2>\n  <p>This website is designed to be accessible to all users...</p>\n  \n  <h3>Features</h3>\n  <ul>\n    <li>Keyboard Navigation: ...</li>\n    <li>Screen Reader Support: ...</li>\n    <li>Text Sizing: ...</li>\n    <li>Color Contrast: ...</li>\n  </ul>\n  \n  <p>For accessibility issues: <a href=\"#contact\">contact us</a></p>\n</section>\n`\n\n### 5.2 Linking from Footer\n\n`html\n<footer>\n  <a href=\"#accessibility-statement\">Accessibility Statement</a>\n</footer>\n`\n\n---\n\n## 6. Testing Checklist\n\n### 6.1 Automated Testing\n\n`\nTools:\n- WAVE: https://wave.webaim.org/\n- Axe DevTools: https://www.deque.com/axe/devtools/\n- Lighthouse: Built into Chrome DevTools\n\nSteps:\n1. Open site in Chrome\n2. Press F12 → Lighthouse tab\n3. Click \"Analyze page load\"\n4. Review accessibility score (should be 90+)\n5. Fix issues listed\n`\n\n### 6.2 Manual Testing\n\n`\nKeyboard Navigation:\n1. Close mouse/trackpad\n2. Press Tab repeatedly\n3. Verify all interactive elements receive focus\n4. Verify focus is visible\n5. Verify logical tab order\n\nScreen Reader (Chrome):\n1. Install ChromeVox extension\n2. Press Alt+Shift+Z to enable\n3. Navigate with arrow keys\n4. Verify all content is read\n5. Verify alt text is read for images\n\nColor Contrast:\n1. Use: https://contrast-ratio.com/\n2. Test text vs background\n3. Ensure 4.5:1 for regular text\n4. Ensure 3:1 for large text\n`\n\n### 6.3 Mobile Testing\n\n`\n1. Open on iPhone/Android\n2. Test zoom to 200%\n3. Verify text remains readable\n4. Test touch targets (44x44px minimum)\n5. Test form inputs are accessible\n`\n\n---\n\n## 7. Metadata Validation\n\n### 7.1 Dublin Core\n\n`\nValidator: https://www.dublincore.org/specifications/dublin-core/dcmi-terms/\n\nValidation Steps:\n1. Check all required elements present\n2. Verify date format (YYYY-MM-DD)\n3. Ensure identifiers are unique\n4. Check rights/license included\n`\n\n### 7.2 Schema.org JSON-LD\n\n`\nValidator: https://validator.schema.org/\n\nSteps:\n1. Copy JSON-LD block\n2. Paste into validator\n3. Click \"Validate\"\n4. Review errors/warnings\n5. Fix any issues\n`\n\n### 7.3 Citation Metadata\n\n`\nGoogle Scholar:\n1. Search for your research\n2. If it appears with metadata → success\n3. If not, submit at: https://scholar.google.com/intl/en/scholar/about.html\n\nResearchGate:\n1. Create profile\n2. Add research\n3. Check if metadata auto-imported\n4. Manually add if needed\n`\n\n---\n\n## 8. Common Errors & Fixes\n\n### Error: \"DOI not resolving\"\n\n**Cause:** DOI not registered properly \n**Fix:**\n1. Check DOI format (should be: 10.XXXX/XXXXX)\n2. Test at: https://doi.org/[YOUR_DOI]\n3. If 404: Register at OSF or Zenodo first\n\n### Error: \"Metadata not showing in Google Scholar\"\n\n**Cause:** Missing or incorrect citation metadata \n**Fix:**\n1. Validate metadata in WAVE or Axe\n2. Check meta tag format is correct\n3. Wait 1-2 weeks for indexing\n4. Manually submit to Google Scholar\n\n### Error: \"ARIA errors in accessibility audit\"\n\n**Cause:** Incorrect ARIA usage \n**Fix:**\n1. Run Axe DevTools\n2. Read error descriptions\n3. Reference: https://www.w3.org/WAI/ARIA/apg/\n4. Fix incorrect roles/labels\n\n### Error: \"Skip link not visible\"\n\n**Cause:** CSS display: none or visibility: hidden \n**Fix:**\n`css\n.skip-link {\n  position: absolute;\n  top: -40px; /* Off-screen, not display:none */\n}\n\n.skip-link:focus {\n  top: 0; /* Becomes visible on Tab */\n}\n`\n\n---\n\n## 9. Best Practices\n\n### DO:\n- ✅ Use semantic HTML (nav, section, article, footer)\n- ✅ Include alt text on all images\n- ✅ Use ARIA labels for custom elements\n- ✅ Test with real assistive technology\n- ✅ Keep metadata updated\n- ✅ Validate regularly\n- ✅ Document compliance\n- ✅ Get user feedback\n\n### DON'T:\n- ❌ Use div/span for everything\n- ❌ Skip alt text (use \"image\" or leave empty)\n- ❌ Rely on color alone for information\n- ❌ Use display:none for skip links\n- ❌ Have metadata in images/PDFs only\n- ❌ Assume standards are optional\n- ❌ Test only with your own browser\n- ❌ Never update metadata\n\n---\n\n## 10.
